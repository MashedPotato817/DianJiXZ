# JY62定位融合调研与预研方案 - 2026-07-11

## 当前结论

JY62适合放在本车上做姿态、角速度、航向辅助，不适合单独承担完整定位。后续定位应采用“编码器里程计为主、JY62航向为辅、灰度/CCD/视觉做场地约束修正”的组合。

当前工程已经具备接入基础：

- 主控为 TI MSPM0G3507。
- 已有差速车、TB6612、电机闭环、灰度/CCD巡线逻辑。
- `UART_1` 已配置为 PB6 TX / PB7 RX / 9600bps，和JY62默认串口速率匹配。
- 现有 `Control/uart_callback.c` 已有UART接收中断/DMA结构，可作为JY62接收框架参考。

## 网上案例学习

### 1. 维特官方标准协议与SDK

维特官方SDK资料提供了标准协议解析思路。JY62常见数据帧为：

- `0x55 0x51`：加速度
- `0x55 0x52`：角速度
- `0x55 0x53`：角度

每帧通常为11字节，最后一字节为校验和。换算关系：

- 加速度：`raw / 32768 * 16g`
- 角速度：`raw / 32768 * 2000 deg/s`
- 角度：`raw / 32768 * 180 deg`

资料来源：

- [WITMOTION STM32 SDK Quick Start](https://wit-motion.gitbook.io/witmotion-sdk/wit-standard-protocol/sdk/stm32_sdk-quick-start)
- [WIT Standard Communication Protocol PDF](https://cdn.robotshop.com/rbm/f83835f4-5e29-4ee0-9cc2-e49300031503/6/681c0521-11c2-4e47-99b0-d336be43f142/8ab75072_wit-standard-communication-protocol.pdf)
- [JY62维特官方产品页](https://wit-motion.cn/proztmz/53.html)

### 2. GitHub上的JY62单片机项目

GitHub代码检索能看到很多电赛或小车项目里单独封装了 `jy62.c/jy62.h`，常见做法是：

- 单独维护一个JY62接收缓冲区。
- 按字节输入状态机找 `0x55` 帧头。
- 校验11字节帧。
- 解出角速度和欧拉角。
- 对外只暴露 `yaw/pitch/roll/gz` 等少量稳定变量。

可参考项目：

- [lanniny/Wheel-fighting - jy62.c](https://github.com/lanniny/Wheel-fighting/blob/master/Core/Src/jy62.c)
- [SK-Wang21/EDC2022 - jy62.h](https://github.com/SK-Wang21/EDC2022/blob/master/Core/Inc/jy62.h)
- [Grange007/EDC25 - jy62.c](https://github.com/Grange007/EDC25/blob/dev/Core/Src/jy62.c)
- [HaiyunZhangRobotics/UART_IMU_STM32F091](https://github.com/HaiyunZhangRobotics/UART_IMU_STM32F091)

这些项目大多是STM32 HAL工程，但协议层和算法思路与MSPM0无关，适合移植到本车。

### 3. ROS/机器人IMU项目

ROS项目里经常把WitMotion IMU作为 `imu` 话题发布，再交给里程计/滤波器融合。这对我们有启发：低层单片机不必一开始做复杂大地图，只要稳定输出 `yaw`、`gyro_z`、车轮距离和状态，后续再上位机或主控内部融合。

可参考：

- [witmotion_ros related code search](https://github.com/search?q=witmotion+ros+imu&type=code)
- [hosogaya/witmotion_imu](https://github.com/hosogaya/witmotion_imu)

## 适合本车的落地路线

### 第一阶段：只读JY62，做调试显示

目标是确认接线、帧解析、安装方向。

硬件建议：

- JY62 VCC接模块要求电压，确认是3.3V还是5V版本。
- GND必须和MSPM0共地。
- JY62 TX接 MSPM0 UART1 RX，也就是PB7。
- JY62 RX接 MSPM0 UART1 TX，也就是PB6；如果只读取数据，RX可以暂时不接。
- 模块尽量固定在车体中心附近，X轴朝车头，Z轴垂直向上，避免软线拉扯导致震动。

软件建议：

- 新增 `Hardware/jy62.c` 和 `Hardware/jy62.h`。
- 中断收到的每个字节调用 `JY62_InputByte(uint8_t b)`。
- 解析成功后更新 `JY62_Data` 全局结构体。
- OLED或串口打印 `yaw/pitch/roll/gz/valid_count/error_count`。

验收标准：

- 静止时角度不乱跳。
- 手动转动车体，yaw方向与实车旋转方向一致。
- 长时间静止5分钟，yaw漂移可记录但不能出现帧错乱。

### 第二阶段：航向保持

目标是在直线巡线、十字路口直冲、丢线找线时减少跑偏。

可做功能：

- 十字路口直冲时记录进入前 `yaw_ref`，直冲期间加一个小的航向修正项。
- 全白找线时优先使用JY62角速度限制旋转速度，避免原地转过头。
- 直线段中，如果灰度偏差很小但左右轮编码器不平衡，用JY62修正 `Move_Z`。

建议控制形式：

```c
heading_error = normalize_angle(yaw_ref - jy62.yaw_deg);
Move_Z += heading_kp * heading_error - gyro_kd * jy62.gz_dps;
```

这一步不改变当前巡线主逻辑，只在特定状态下叠加小修正。

### 第三阶段：编码器里程计 + JY62航向

目标是维护一个低成本位姿估计：

```text
x += distance * cos(theta)
y += distance * sin(theta)
theta = jy62_yaw 或 encoder_theta 与 jy62_yaw融合
```

编码器负责距离，JY62负责角度。灰度线、十字路口、已知赛道节点可以作为“校准点”：

- 识别到十字路口：修正当前位置到节点附近。
- 长直线巡线稳定：将横向偏差约束到黑线附近。
- 定点停车：用编码器距离为主，JY62保证车身方向。

### 第四阶段：赛题扩展接口

赛题未公布前，建议预留这些接口：

- `car_pose.c/h`：维护 `x, y, theta, speed, valid`。
- `jy62.c/h`：只负责IMU解析和角度零点。
- `encoder_odom.c/h`：只负责左右轮距离、速度、里程。
- `navigation_state.c/h`：负责十字、拐角、节点、任务状态。
- `debug_log.c/h`：统一串口/OLED调试输出。

这样赛题出来后，不管是巡线、定位、寻物、避障、视觉识别，都能比较快拼起来。

## 风险点

- JY62六轴版本没有磁力计，yaw会随时间漂移，不能当绝对方向长期使用。
- 电机PWM和电源纹波可能干扰串口，JY62线尽量短，GND可靠，必要时加屏蔽或远离电机线。
- 不要在定时中断里做大量浮点打印和阻塞串口发送。
- 如果UART1继续给蓝牙使用，就需要改用UART0或重新规划蓝牙/JY62/K210的串口分配。
- 只靠加速度二次积分做定位不可取，漂移会非常快。

## 下一步建议

1. 先固定JY62安装方向，拍照记录接线和安装位置。
2. 在当前仓库新建JY62解析模块，不动现有巡线主逻辑。
3. OLED显示 `yaw`、`gz`、帧计数和错误计数。
4. 跑静止、原地转圈、直线巡线三组测试，记录漂移和方向。
5. 再把JY62用于十字直冲航向保持和全白找线限速。
