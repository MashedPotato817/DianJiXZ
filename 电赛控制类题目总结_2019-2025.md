## 一句话结论

近几年电赛控制题已经从“让车跑起来、让电机转起来”发展为“感知、定位、运动控制、任务状态机、通信和系统可靠性综合考核”。小车仍是高频载体，但题目越来越强调视觉、声学/超声定位、自动瞄准、避障、巡查、协同作业和多传感器融合。

对当前项目来说，`MSPM0 + TB6612 小车底盘 + 灰度模块 + K210/热成像/超声` 的方向是比较贴近近年控制题趋势的。

## 控制类题目的常见评分结构

电赛题面通常会把任务分成“基本要求”和“发挥部分”。控制类题目虽然每年细节不同，但评分逻辑高度相似。

### 1. 基本要求

基本要求通常考察系统能否稳定完成核心任务，常见占比约为 50 分左右，具体以当年题面为准。

常见评分点：

| 评分点 | 主要看什么 | 常见扣分原因 |
|---|---|---|
| 硬件系统完整性 | 主控、电源、传感器、执行机构能稳定工作 | 接线不牢、供电压降、干扰导致复位 |
| 基础运动/执行 | 小车能走、云台能转、灯光/电机/泵等执行器能响应 | 执行动作不一致、方向反、速度不稳 |
| 基础感知 | 能识别线、障碍、目标、坐标、温度/光照等输入 | 传感器安装差、阈值漂移、误检漏检 |
| 基础闭环 | 能根据误差自动修正位置、速度、角度、亮度等 | 只有开环动作，无法纠偏 |
| 指定任务完成 | 按题目要求完成规定路径、命中、定位、送达等 | 超时、偏差大、未到指定区域 |
| 人机交互/显示 | 能显示关键状态、参数、结果 | 现场无法判断系统状态 |

### 2. 发挥部分

发挥部分通常用于区分高分队伍，常见占比约为 50 分左右。

常见评分点：

| 评分点 | 主要看什么 | 高分做法 |
|---|---|---|
| 精度提升 | 定位误差、命中误差、停车误差、测量误差 | 标定、滤波、闭环修正、多次采样 |
| 速度提升 | 更短时间完成巡线、避障、追踪、配送 | 分段速度、弯道减速、任务并行 |
| 稳定性提升 | 多次测试成功率 | 状态机、异常恢复、抗干扰设计 |
| 自动化程度 | 是否减少人工干预 | 上电自检、自动校准、自动复位 |
| 复杂场景适应 | 光照变化、障碍变化、目标移动、噪声 | 多传感器融合、动态阈值 |
| 扩展功能 | 题目允许的附加功能 | 数据记录、路径优化、远程监控 |

### 3. 设计报告

设计报告通常单独占一部分分值，或作为总评分的重要参考。控制类报告不应该只堆代码截图，重点要说明“为什么这样设计，如何保证稳定”。

报告建议包含：

- 系统方案框图：主控、传感器、执行机构、电源、通信接口。
- 控制流程图：状态机、任务流程、异常处理。
- 关键算法：PID、滤波、定位解算、路径规划、图像识别逻辑。
- 参数整定方法：速度、阈值、PID 参数、传感器标定。
- 测试数据：成功率、误差、耗时、重复性。
- 问题与改进：现场可能故障、备选方案、抗干扰处理。

### 4. 现场测试常见规则

控制类题目的现场测试很看稳定性。常见规则包括：

- 每项任务通常给有限测试次数，取最好成绩或按完成情况计分。
- 超时通常不得分或扣分。
- 人工干预通常扣分，严重时该项无效。
- 系统复位、脱离场地、碰撞、掉落、误动作通常会扣分。
- 题目要求的外形尺寸、电源方式、指定器件、通信方式必须满足。
- 现场光照、地面摩擦、供电、噪声可能和实验室不同。

## 各年份题目分析

### 2019 年：小车与基础自动控制仍是主线

2019 年控制相关题目的特征是：任务物理性强，机械结构、电源和基础闭环决定成败。

#### A 电动小车动态无线充电系统

关键词：

- 小车运动控制
- 无线供电/充电
- 对准与停车
- 电源管理

考点：

- 小车能否沿指定路线稳定运行。
- 接收端与发射端能否保持有效耦合。
- 动态运动中供电是否稳定。
- 充电、储能、负载切换是否可靠。

备赛启发：

- 控制题经常和电源题混合。
- 小车电源、驱动电源、传感器电源必须分层设计。
- 电机噪声会影响 ADC、通信和定位。

#### B 巡线机器人

关键词：

- 灰度/视觉循迹
- 弯道控制
- 停车定位
- 路径判断

考点：

- 直线速度和弯道稳定性。
- 交叉口、标志点、终点识别。
- 路径偏离后的纠正能力。
- 多次运行一致性。

备赛启发：

- 灰度模块是入门，但高分要做速度规划和状态机。
- 不要只写“看到黑线左转/右转”，要做偏差量和比例控制。
- 编码器对直行、定距停车和转角很重要。

#### H/J 模拟电磁曲射炮

关键词：

- 角度控制
- 发射能量控制
- 目标定位
- 命中精度

考点：

- 发射机构一致性。
- 角度调节精度。
- 目标距离和角度估计。
- 多次命中稳定性。

备赛启发：

- 控制不只小车，云台、炮塔、机械臂、投射机构都属于控制。
- 机械重复性比算法更重要。

#### K 简易多功能液体容器

关键词：

- 液位检测
- 温度/流量控制
- 执行机构控制
- 过程控制

考点：

- 液位或温度能否到达设定值。
- 控制过程是否超调。
- 多功能切换是否可靠。

备赛启发：

- 过程控制题强调稳定、精度和安全保护。
- PID、滞回控制、阈值控制都可能用到。

### 2021 年：任务型机器人和闭环调节增强

2021 年控制题开始明显强调“任务流程”。小车不只是跑路线，还要完成配送、识别、能量管理等任务。

#### F 智能送药小车

关键词：

- 自主小车
- 任务点识别
- 路径规划
- 物品投递/到达确认

考点：

- 能否识别或到达指定病房/站点。
- 路径选择是否正确。
- 停车精度和任务执行可靠性。
- 是否能处理多个任务点。

备赛启发：

- 要做清晰状态机：待命、识别任务、寻路、到站、执行、返回。
- 显示屏/串口日志很重要，现场能快速判断当前状态。

#### G 植保飞行器

关键词：

- 飞行控制
- 定位/识别
- 区域作业
- 安全控制

考点：

- 飞行稳定性。
- 作业区域识别。
- 定点、定高、路径执行。
- 安全保护。

备赛启发：

- 飞行器题门槛高，机械与安全风险大。
- 传感器融合和保护逻辑很关键。

#### I 具有发电功能的储能小车

关键词：

- 小车运动
- 发电与储能
- 能量转换
- 电源管理

考点：

- 小车运行与发电机构耦合。
- 储能效率和输出能力。
- 运动稳定性与能量采集平衡。

备赛启发：

- 电源管理会影响控制系统可靠性。
- 控制类队伍也要懂基础电源测量和保护。

#### K 照度稳定可调 LED 台灯

关键词：

- 光照检测
- PWM 调光
- 闭环稳定
- 抗环境光干扰

考点：

- 照度设定精度。
- 环境光变化下的稳定性。
- 调节速度和超调。
- 显示与交互。

备赛启发：

- 这是典型单变量闭环控制题。
- 滤波、采样周期、控制周期会直接影响表现。

### 2023 年：感知定位和协同系统成为重点

2023 年题目明显提高了系统综合度：目标跟踪、声学定位、空地协同、低摩擦平台运动都要求把传感、算法和执行结合起来。

#### E 运动目标控制与自动追踪系统

关键词：

- 运动目标
- 自动追踪
- 视觉/传感检测
- 执行机构闭环

考点：

- 目标位置检测。
- 追踪响应速度。
- 跟踪误差。
- 目标运动变化下的稳定性。

备赛启发：

- 摄像头、K210、OpenMV、激光点、云台控制都是常见组合。
- 追踪题要处理延迟，不能只靠简单阈值。

#### F 基于声传播的智能定位系统

关键词：

- 声学测距
- TDOA/TOF
- 坐标解算
- 误差校正

考点：

- 时间测量精度。
- 多点定位算法。
- 坐标误差。
- 抗噪声能力。

备赛启发：

- 定位题核心是标定和误差建模。
- 定时器捕获、同步、滤波比花哨算法更重要。

#### G 空地协同智能消防系统

关键词：

- 空地协同
- 目标识别
- 通信
- 任务执行

考点：

- 空中/地面平台之间的信息传递。
- 火源或目标识别。
- 地面平台到达与执行。
- 协同流程是否稳定。

备赛启发：

- 通信协议要简单可靠。
- 多设备系统必须有主从状态和超时处理。

#### I 气垫悬浮车

关键词：

- 低摩擦运动
- 方向控制
- 轨迹控制
- 姿态扰动

考点：

- 悬浮稳定性。
- 低摩擦情况下的转向和制动。
- 指定路径完成度。

备赛启发：

- 机械结构和气流控制决定上限。
- 低摩擦系统惯性大，控制要提前量。

### 2025 年：识别、定位、瞄准、巡查、避障进一步融合

2025 年控制类题目更像“小型智能系统”。很多任务不再单独考电机，而是考“感知后如何动作”。

#### E 简易自行瞄准装置

关键词：

- 自动瞄准
- 目标识别/定位
- 云台或执行机构控制
- 命中/对准精度

考点：

- 能否识别目标位置。
- 瞄准速度和精度。
- 多次对准一致性。
- 执行机构机械回差和抖动。

备赛启发：

- 云台 PID、图像坐标到角度映射、舵机/步进电机控制要练。
- 机械结构刚性会直接影响得分。

#### H 野生动物巡查系统

关键词：

- 巡查
- 图像识别
- 目标记录
- 移动平台/固定平台任务

考点：

- 能否完成巡查路线。
- 能否识别指定目标。
- 能否记录或显示识别结果。
- 误识别和漏识别情况。

备赛启发：

- K210/视觉模块适合这类题。
- 数据记录和结果显示是容易拿分的部分。

#### I 非接触式控制盘

关键词：

- 手势/近距离检测
- 交互控制
- 多通道输入
- 响应速度

考点：

- 非接触动作识别准确率。
- 响应延迟。
- 多种控制命令区分。
- 抗误触能力。

备赛启发：

- ToF、红外、霍尔、电容、超声都可能成为输入方案。
- 阈值、滤波和状态判定比传感器堆料更重要。

#### J 超声信标定位系统

关键词：

- 超声测距
- 信标定位
- 坐标解算
- 误差控制

考点：

- 多基站/多信标测距。
- 坐标解算准确性。
- 抗反射、抗干扰。
- 标定流程。

备赛启发：

- 需要熟悉定时器输入捕获、互相关/阈值检测、温度补偿。
- 现场环境反射会严重影响超声。

#### K 自动避障小车

关键词：

- 小车底盘
- 障碍检测
- 路径规划
- 速度控制

考点：

- 能否识别障碍。
- 避障路径是否正确。
- 是否碰撞。
- 完成时间和稳定性。

备赛启发：

- 超声/红外/ToF/视觉都可用。
- 高分关键是“能稳定绕过去”，不是传感器越多越好。

## 控制类题目技术能力拆解

### 底层硬件能力

必须掌握：

- 电机驱动：TB6612、DRV8833、L298N 等。
- PWM：频率、占空比、死区、方向控制。
- 编码器：测速、计数、定距、转角估算。
- 舵机/云台：角度控制、限位保护、回差处理。
- 电源：主控 3.3 V、传感器 5 V、动力电源隔离和共地。

常见风险：

- 电机启动电流导致主控复位。
- 串口通信受电机干扰。
- 传感器供电不稳导致阈值漂移。
- 地线不共地导致通信异常。

### 传感器能力

常见传感器与用途：

| 传感器 | 适合题型 | 注意点 |
|---|---|---|
| 灰度模块 | 巡线、路径识别 | 光照变化、安装高度 |
| 编码器 | 定距、测速、转角估计 | 打滑会造成累计误差 |
| 超声 | 避障、测距、定位 | 反射、多径、盲区 |
| ToF | 近距离避障、手势 | 视场角、强光干扰 |
| IMU | 姿态、角速度、转向辅助 | 零偏、漂移 |
| K210/OpenMV | 目标识别、巡查、瞄准 | 光照、模型训练、串口协议 |
| 热成像 | 人体/热源检测、巡查 | 帧率、串口带宽、阈值标定 |

### 控制算法能力

最低要求：

- PWM 开环控制。
- 比例巡线。
- 简单 PID 调速。
- 阈值判断和消抖。
- 状态机任务流程。

高分能力：

- 编码器闭环速度控制。
- 直行偏航修正。
- 路径分段速度规划。
- 目标追踪 PID。
- 多传感器融合。
- 超时保护和异常恢复。

### 软件架构能力

控制类题最怕“所有逻辑塞进 while(1)”。推荐结构：

```text
初始化
  -> 传感器自检
  -> 等待启动
  -> 任务状态机
      -> 寻线/巡航
      -> 识别目标/路口/障碍
      -> 定位/转向/停车
      -> 执行动作
      -> 记录结果/返回
  -> 异常处理
```

建议模块划分：

| 模块 | 职责 |
|---|---|
| `motor` | PWM、方向、电机限幅 |
| `encoder` | 计数、测速、定距 |
| `line_sensor` | 灰度采样、偏差计算 |
| `distance_sensor` | 超声/ToF 读取与滤波 |
| `vision` | K210/热成像串口解析 |
| `control` | PID、速度规划、转向控制 |
| `task` | 状态机和任务流程 |
| `debug` | 串口打印、OLED 显示 |

## 评分导向下的备赛优先级

### 第一阶段：先保证基本分

目标：任何时候都能跑、能停、能测。

必做：

- 小车直行不明显跑偏。
- 能稳定低速巡线。
- 能定距停车。
- 能识别障碍并停车。
- 串口或屏幕能显示当前状态。
- 每次上电行为一致。

验收标准：

- 连续运行 10 次，至少 8 次完成基础路线。
- 电池电压下降后仍能运行。
- 手动搬动或轻微偏离后能恢复。

### 第二阶段：提高发挥分

目标：更快、更准、更稳。

必做：

- 编码器速度闭环。
- 弯道减速、直道加速。
- 路口/任务点状态机。
- 目标识别串口协议稳定。
- 关键参数可现场调整。

验收标准：

- 同一路线耗时稳定。
- 停车误差可测、可调。
- 识别误检率可接受。

### 第三阶段：现场鲁棒性

目标：换场地也能跑。

必做：

- 灰度阈值自动或半自动标定。
- 摄像头曝光/阈值预案。
- 电源抗干扰。
- 所有线束固定。
- 备份程序和参数。

验收标准：

- 改变光照后仍能完成基础任务。
- 电机大电流动作时主控不复位。
- 通信断帧后能恢复。

## 针对当前项目的建议

当前项目已有或计划使用：

- MSPM0G3507 主控。
- TB6612 电机驱动。
- 编码器。
- 八路灰度模块。
- K210 视觉模块。
- HTPA32 热成像模块。

建议路线：

1. 先把 `TB6612 + 编码器` 做成稳定底盘。
2. 再做灰度巡线和定点停车。
3. 然后接 K210，完成目标识别或标志识别。
4. 最后接热成像或超声，作为人体/热源/障碍检测扩展。
5. 所有外设统一走清晰串口协议，避免临场调不出来。

小车底层建议指标：

| 项目 | 建议目标 |
|---|---|
| PWM 控制 | 左右轮方向正确，占空比可调 |
| 编码器测速 | 10 ms 或 20 ms 周期更新 |
| 直行控制 | 左右轮速度差闭环修正 |
| 定距停车 | 误差先做到 5-10 cm，再继续优化 |
| 巡线 | 先低速稳定，再提速 |
| 调试输出 | 至少打印状态、速度、传感器值 |

热成像接入建议：

- 不建议一开始解析完整 32x32 热图。
- 优先用模块摘要输出：最高温、最低温、中心温、平均温、最高温坐标。
- 用最高温坐标判断热源方向，用最高温数值判断是否发现目标。
- 热成像数据帧率低，适合“发现目标/确认目标”，不适合高速闭环巡线。

## 常见失分点清单

| 失分点 | 原因 | 规避方法 |
|---|---|---|
| 小车跑偏 | 左右电机差异、轮胎摩擦不同 | 编码器闭环、速度标定 |
| 弯道冲出 | 速度太高、控制滞后 | 弯道减速、偏差限幅 |
| 停车不准 | 只按延时停车 | 编码器定距、目标点二次校正 |
| 目标识别不稳 | 光照变化、阈值固定 | 自动曝光/阈值标定、增加补光 |
| 通信丢包 | 电机干扰、协议无帧头校验 | 帧头、长度、校验、超时 |
| 现场复位 | 电源压降、地线布置差 | 动力和主控分供电、共地、滤波 |
| 调试困难 | 无显示、无日志 | OLED/串口输出关键状态 |
| 临场参数不合适 | 实验室环境和现场不同 | 参数可调、保存默认值 |

## 备赛训练题单

按优先级练：

1. 小车直行 1 m、2 m、3 m 定距停车。
2. 原地左转/右转 90 度、180 度。
3. 灰度循迹跑直线、弯道、十字路口。
4. 路口计数并在指定路口停车。
5. 遇障停车、绕障、回到路线。
6. K210 识别颜色块/二维码/数字，并通过 UART 发给主控。
7. 热成像识别热源方向，主控根据方向转向。
8. 小车完成“出发-识别-到点-执行-返回”的完整状态机。

## 推荐代码框架

```c
int main(void)
{
    board_init();
    motor_init();
    encoder_init();
    line_sensor_init();
    vision_init();

    task_state = TASK_WAIT_START;

    while (1) {
        sensor_update();
        vision_update();
        task_update();
        control_update();
        debug_update();
    }
}
```

中断里只做短任务：

```c
void timer_10ms_irq(void)
{
    encoder_sample();
    control_tick = 1;
}

void uart_rx_irq(void)
{
    push_byte_to_ring_buffer();
}
```

不要在中断里做：

- 大量 `printf`
- 长延时
- 图像解析
- 复杂路径规划
- 阻塞等待串口

## 结论

从 2019 到 2025，控制类题目的核心能力可以概括为：

```text
稳定硬件 + 可靠传感 + 闭环控制 + 任务状态机 + 现场鲁棒性
```

如果只能押一个方向，小车平台仍然最值得练；如果想贴近近年趋势，小车上应继续叠加视觉、超声/ToF、热成像、通信协议和任务状态机。最终目标不是“某个模块能用”，而是“系统能在现场连续完成任务”。
