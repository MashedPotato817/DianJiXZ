# 智能送药小车（F题）实现路线与任务拆解

## 1. 先定边界

第一目标是完成**单车基本要求**，即从药房出发，向任意指定的 1～8 号病房送药并自动返回。双车协同建立在单车连续稳定运行以后，不与单车基础功能同时开发。

本题红线是行驶引导线，黑色数字纸张是目的地标识；二者不要混为一个识别问题：

- 底盘负责跟随红线、通过路口、停车和原路返回；
- K230 只负责识别数字标号，并把目标号或当前门牌号发送给 MSPM0；
- MSPM0 负责路线决策、装卸检测、灯光和电机状态机。

> 现有 `WHEELTEC_C07A_CAR` 的 8 路灰度程序按黑线设计。必须先在白底红胶带和实际灯光下实测其输出；不能仅凭“红色较深”就假定能巡线。

## 2. 场地路线分解

把场地抽象成一条主干和三层决策点，而不是把每个房间写成一套运动脚本。

```text
药房 P
  │ 主干上行
近端路口 N ── 左支：1 号门       右支：2 号门
  │ 主干上行
中部路口 M ── 左支：3/4 号候选    右支：3/4 号候选
  │ 主干上行
远端路口 R ── 左远端：5/7 候选    右远端：6/8 候选
```

其中 1、2 号固定；3～8 的数字位置会随机改变。因此：

1. 药房先识别手持目标号 `target_room`；
2. 到每一个路口的“看牌点”减速或短暂停车，K230 读取左右门牌及其画面横坐标；
3. 若目标号在当前可达分支，进入该分支；否则沿主干继续；
4. 进入目标门口区域后停车；
5. 返回时不再识别数字，而是记录/回放出发时的路口序列，反向走回 P。

### 2.1 路段与事件

| 路段/位置 | 底盘动作 | 视觉任务 | 状态机事件 |
| --- | --- | --- | --- |
| P：药房 | 停车、绿灯或待机灯 | 识别手持目标号 | `TARGET_LOCKED`、`LOAD_OK` |
| P→N | 红线巡线 | 无 | `NODE_N` |
| N：近端路口 | 减速、看左右 | 识别 1/2 | `BRANCH_LEFT` / `BRANCH_RIGHT` / `GO_STRAIGHT` |
| N→M | 红线巡线 | 无 | `NODE_M` |
| M：中部路口 | 减速、看左右 | 识别当前 3/4 门牌 | 同上 |
| M→R | 红线巡线 | 无 | `NODE_R` |
| R：远端路口 | 减速、看左右 | 识别当前 5～8 门牌 | 同上 |
| 目标门口 | 停车、亮红灯 | 可选二次核对目标号 | `ARRIVED`、`UNLOAD_OK` |
| 回程 | 按已记录路口反向巡线 | 不需要 | `HOME` |

`NODE_N/M/R` 不建议只用“经过几厘米”触发：应优先由路口特征（多路传感器同时见线、门口虚线等）触发，编码器里程只用于超时保护和门口微调。

## 3. 单车状态机

```text
IDLE
 └─识别目标号→ WAIT_LOAD
WAIT_LOAD
 └─检测到约 200 g 装载且稳定→ OUTBOUND
OUTBOUND
 └─到路口→ READ_SIGN → 选左/右/直行 → OUTBOUND
 └─目标门口确认→ ARRIVED_WAIT_UNLOAD
ARRIVED_WAIT_UNLOAD
 └─红灯亮、车轮锁止；检测到卸载→ RETURN
RETURN
 └─按返程记录反向通过路口→ HOME
HOME
 └─绿灯亮、车轮锁止；复位→ IDLE
```

实现时用 `enum` 定义状态，用独立变量保存：

- `target_room`：药房锁定的目标号，范围 1～8；
- `route_stack[]`：去程实际选择的 `LEFT/RIGHT/STRAIGHT`，回程倒序执行；
- `current_node`：`P/N/M/R`；
- `load_present`：装药检测结果；
- `sign_result`、`sign_fresh`：K230 最近一次门牌识别结果及新鲜度；
- `state_enter_ms`：状态超时保护；任何异常、丢线超时或识别超时均停车报警，不允许盲走。

## 4. 分阶段实现与验收

每阶段通过后再进入下一阶段；每次只增加一个变量，便于定位问题。

| 阶段 | 要做什么 | 通过标准 |
| --- | --- | --- |
| 0. 赛道标定 | 用实际白底、红线、黑数字、比赛灯光采集传感器和相机数据；量每个看牌点、门口、路口的里程 | 得到红线判定极性/阈值，确认相机能看清 8×6 cm 数字 |
| 1. 红线巡线 | 仅修改现有 `Gray_Mode()` 的线色判定与路口检测；先不接任务状态机 | 空载连续 P→N→P 10 次，无撞墙、无人工扶正 |
| 2. 路口动作 | 实现“进路口减速—直行/左转/右转—重新找到线”；用按键手动指定动作 | N、M、R 每种动作各连续成功 10 次 |
| 3. 装卸检测 | 加装载传感器（称重、微动开关或挡板限位），做去抖与稳定判定 | 加载才启动；卸载后才返回；误触发为 0 |
| 4. K230 数字识别 | 训练/部署 1～8 数字识别；多帧一致锁定，串口传给 MSPM0 | 药房 20 s 内稳定识别；各数字在不同光照、距离下连续 20 次正确 |
| 5. 看牌决策 | 在 N/M/R 的看牌点取得左右候选号码及其画面左右位置，选目标所在支路 | 随机摆放 3～8 后，20 次均能选对支路 |
| 6. 单车闭环 | 接入完整状态机：目标识别、装载、送达、卸载、返回、绿灯 | 近/中/远端各 10 次；去程和回程均低于 20 s |
| 7. 双车协同 | 加无线事件协议、暂停点和避让规则 | 单车稳定后再测发挥部分（1），最后才做发挥部分（2） |

## 5. K230 与 MSPM0 的接口

现有 K230 链路的坐标帧适合色块/目标位置；数字任务建议另加一种明确的数据帧，避免复用 `x/y/area` 字段产生歧义。

```text
K230 → MSPM0：0xAA, TYPE, ROOM, X_L, X_H, CONF, XOR

TYPE = 0x01：药房目标号
TYPE = 0x02：路口左/右候选门牌号
ROOM = 1..8，0 表示未识别
X    = 数字框中心横坐标，用于判定左/右
CONF = 置信度或连续一致帧数
```

MSPM0 只有在 `ROOM` 连续多帧一致、置信度达标且帧未超时时才更新结果。K230 离线或识别超时的策略是**停车等待/报错**，不能按上一帧结果继续开。

## 6. 工程落点

在现有工程中建议新增独立任务模块，而不是将送药逻辑塞进巡线函数：

```text
Control/medicine_task.c/.h   状态机、路线栈、超时和灯光命令
Hardware/load_sensor.c/.h    装载/卸载传感器读取与去抖
Hardware/k230_link.c/.h      K230 数字帧解析（可从已有 K210 链路迁移）
Control/control.c            只保留 5 ms 巡线/电机闭环；任务层给出速度和路口动作
```

先从 `firmware/WHEELTEC_C07A_CAR` 作为底盘基础开始；若要复用 JY62/K230 链路，选择性迁移 `WHEELTEC_C07A_CAR_JY62_GapBridge` 中已验证的模块，不能直接把该目录作为成品工程，因为其 `empty.c` 当前是云台 PWM 演示入口。

## 7. 双车阶段（最后做）

双车只传递**事件**，不要传连续遥控速度：

```text
车 1：ARRIVED → UNLOADED → RETURNING → HOME
车 2：WAIT_START → RUNNING → PAUSED → RUNNING → ARRIVED
```

- 发挥（1）：车 2 在独立、安全的暂停点等待 `UNLOADED`，暂停点与车 1 的红线路径距离不少于 70 cm；
- 发挥（2）：车 2 必须等待车 1 的“取药开始”事件才出发；
- 任何无线超时、对方状态未知、两车距离不满足规则时，默认停车亮黄灯；
- 两车在同一走廊不能错车或超车，因此路线和发车时机要先在纸面上验证。

## 8. 首次实车调试清单

1. 测红线在 8 路传感器上的原始电平，记录白底/红线/黑边界三组数据。
2. 用 K230 在药房识别一张固定的数字纸，确认 UART 帧、校验、MSPM0 显示都正确。
3. 空载低速跑 P→N→P，再逐步扩大到 M、R；每次只开放一个路口动作。
4. 接入装载检测后，验证“未装不走、未卸不回”。
5. 最后才随机粘贴 3～8 号门牌，连续跑完整送返测试并记录失败位置、光照和状态机状态。
