# DianJiXZ

全国大学生电子设计竞赛控制类备赛仓库。这里保存多个可独立演进的固件项目、赛题资料、学习笔记和技术预案；不同目录可能面向不同赛题、硬件组合或验证目标，不应假定它们共用同一套接线、参数或入口程序。

## 从哪里开始

- [固件项目导航](firmware/README.md)：选择与当前任务对应的固件工程。
- [GitHub Pages 导航页](index.html)：浏览学习笔记、技术路线、赛题归档和 2026 预案。
- [学习笔记](docs_learn/README.md)：PWM/PID、巡线、运动学、JY62、K230、VOFA+ 与云台。
- [技术路线](workflow/index.md)：当前硬件状态、技术预案和风险记录。

| `workflow/` | 跨项目的技术路线与任务规划。 |

## 固件项目约定

1. `firmware/` 的每个直接子项目必须维护自己的 `README.md`，写明用途、硬件、构建入口、运行方式和验证范围。
2. 新赛题优先新建独立项目目录；不要为了复用代码直接覆盖已有赛题工程。
3. Keil 的 `Objects/`、`Listings/`、用户工作区文件和 Python 缓存不提交。生成的 SysConfig 文件是否入库，由对应项目 README 说明。
4. 硬件结果以该项目 README 中的“已验证范围”为准；主机端编译通过不等于实车完成。
