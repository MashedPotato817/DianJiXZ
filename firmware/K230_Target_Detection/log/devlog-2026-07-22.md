# 开发日志 2026-07-22

## 今日目标

K230 黑框靶面定位——在复杂场景中稳定检测靶纸矩形框，输出靶心坐标。

## 路线演进（5 次迭代）

### 1. blob 方案 → 失败

- **黑框 blob** `(0,96)`：阈值太宽，全画面被判为黑色，检测框覆盖整帧
- **白纸 blob** `(190,255)`：灯光也是白的，框锁到灯上
- **根因**：灰度阈值没有形状信息，无法区分靶纸和背景

### 2. 预处理 + find_rects() → 可行 (`d210df6`)

调研 2025 电赛 E 题社区方案，基本管线：
```
灰度 → binary(0,80) → invert → find_rects()
```
形状感知——只检四边形，灯光不会误触。开始可用，但掉帧严重。

### 3. 性能优化 → 640→320 (`650ac6e`)

- `mean_pooled(2,2)` 降采样到 320×240 处理，显示保持 640×480
- 后来砍掉 mean_pooled，sensor 直出 QVGA 320×240
- 逐步砍掉 gaussian、erode、dilate，管线精简为 `binary→invert→find_rects`

### 4. 追踪稳定性 → 时序保持 + 距离门控 (`1fdc9df` → `697a7cf`)

- **时序保持** `HOLD_FRAMES=5`：检测短暂丢失时复用上一帧 (`1fdc9df`)
- **连续确认** `detect_streak≥2`：单帧假阳性不激活 hold (`1fdc9df`)
- **距离门控** `MAX_JUMP=60`：跳变超过阈值判定为假阳性，拒绝更新 (`697a7cf`)
- **圆形替代四边形**：圆心=对角线交点，半径=avg(角点距)，视觉更干净 (`697a7cf`)

### 5. cv_lite 硬件加速 → 质的飞跃 (`bd492a9` → `7cfbcd1`)

刷 v1.8 固件后 `cv_lite` 全模块可用。`grayscale_find_rectangles_with_corners()` 替代原生管线：
- 检测连续性 ~100%（原生 ~50%）
- 坐标抖动 ±1-3px（原生 ±5-10px）
- 零假阳性、零丢失帧
- 代码从 3 步预处理简化为单函数调用
