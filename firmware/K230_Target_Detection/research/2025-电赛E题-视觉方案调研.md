## 赛题概述

**2025 年全国大学生电子设计竞赛 E 题「简易自行瞄准装置」**，要求设计小车+瞄准模块。核心视觉任务：识别矩形靶纸、提取靶心坐标、控制激光瞄准。

### 官方靶纸规格（与我们的差异）

| 特征 | 官方 E 题 | 我们的靶纸 |
|------|----------|-----------|
| 外框 | 1.8cm 宽黑色胶带 | 18.5mm 宽黑色外框 |
| 靶心 | 红色油性笔点 ≤0.1cm | 红色中心点 |
| 同心圆 | **红色**，r=2/4/6/8/10cm | **黑色**，直径 40mm 整数倍 |
| 纸幅 | A4 (210×297mm) | 209.5×296mm（≈A4） |

差异不影响视觉方案选择——核心都是「黑色外框 + 白底 + 同心圆 + 红点」。

## 社区主流方案对比

### 方案 A：预处理 + find_rects()（✅ 推荐，改动最小）

**原理**：灰度 → 二值化提取暗区（0~80）→ 反色使黑框变白色前景 → 腐蚀去同心圆干扰 → 膨胀连接黑框断边 → `find_rects()` 检测四边形。

**优点**：
- 纯 CanMV 原生 API，无需换固件或训练模型
- 形状感知——灯光再亮也不可能是矩形
- 社区大量成功案例

**缺点**：
- 需要调参（阈值、形态学核大小）
- 帧率一般（~15fps @ 640×480）

**关键代码骨架**（来自多个社区方案整合）：
```python
# 预处理
img = img.gaussian(1)               # 去噪
img.binary([(0, 80)])               # 二值化：暗区→白
img.invert()                         # 反色：黑框→白色前景
img.erode(1)                         # 去同心圆细线
img.dilate(1)                        # 连接黑框断边

# 矩形检测
for r in img.find_rects(threshold=5000):
    corners = r.corners()            # 四个角点
    cx = sum(p[0] for p in corners) // 4
    cy = sum(p[1] for p in corners) // 4
```

**参考来源**：
- CSDN: [K230 矩形检测+透视变换+同心圆](https://blog.csdn.net/2402_82618712/article/details/149888692)
- 01Studio Wiki: [find_rects 教程](https://wiki.01studio.cc/en/docs/canmv_k230/machine_vision/image_detection/find_rects/)
- 多个 2025 电赛 B 站开源方案

**关键参数**：

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 二值化阈值 | `(0, 80)` | 暗区上限；光线暗时降到 `(0, 60)` |
| erode 次数 | `1` | 去细线噪点（同心圆线宽 < 黑框宽） |
| dilate 次数 | `1` | 连接黑框可能存在的断边 |
| find_rects threshold | `5000~8000` | 低于默认 10000，提高黑框灵敏度 |
| 面积筛选 | `1500~55000` | 过滤过小/过大矩形 |
| 长宽比 | `0.5~0.95` | 靶纸 209.5/296≈0.708 |

### 方案 B：cv_lite 硬件加速（性能最佳）

**原理**：K230 的 cv_lite 模块底层用 OpenCV 实现（Canny 边缘检测 → 多边形拟合 → 矩形筛选），40-60fps。

```python
import cv_lite
rects = cv_lite.grayscale_find_rectangles_with_corners(
    [height, width], img_np,
    canny_thresh1=50, canny_thresh2=150,
    approx_epsilon=0.04, area_min_ratio=0.005,
    max_angle_cos=0.3, gaussian_blur_size=3
)
```

**缺点**：需要刷 01Studio 的 daily build 固件。当前 CanMV `0295137e6 (2025-09-20)` 大概率不含 cv_lite。需验证 `import cv_lite` 是否可用。

**参考**：
- [cv_lite API 手册](https://www.kendryte.com/k230_canmv/en/main/api/cv_lite/cv_lite.html)
- [01Studio cv_lite 介绍](https://wiki.01studio.cc/en/docs/canmv_k230/machine_vision/cv_lite/intro/)

### 方案 C：YOLOv5 AI 检测（鲁棒性最强，工作量大）

训练一个检测靶纸的目标检测模型。MaixCAM 平台使用的方案，适合背景极其复杂、遮挡频繁的场景。

**缺点**：需收集标注数据、训练、转换为 KModel，README 已指出"现阶段不需要训练模型"。

**参考**：GitHub [Black-Cyan/NUEDC-2025](https://github.com/Black-Cyan/NUEDC-2025)

## 为什么之前的 blob 方案行不通

| 尝试 | 失败原因 |
|------|---------|
| 黑色连通域 `(0,96)` | 阈值太宽，整个画面被判定为黑色；阈值收紧后黑框线太细，连通域不完整 |
| 白色连通域 `(190,255)` | 灯光也是白色的，靶纸白底与灯光无法区分 |

**根本原因**：单通道灰度阈值没有形状信息。灯光、白墙、阴影都能通过灰度筛选。

**为什么 find_rects 能解决**：它检测的是**四边形轮廓**。灯光再亮、阴影再大，只要不是矩形就不会被检出。
