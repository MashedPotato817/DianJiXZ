## 1. 本次观察结论

检查 `pic/` 中 2026-07-23 的三张实拍帧及 `log.md` 后确认：当前画面叠加的圆、十字和四角落在窗框/窗格上，而不是可确认的靶纸。当前系统把背景中与 A4 靶纸长宽比相近的窗格持续判为 `detect`，因此“连续检测”不等于“正确识别靶纸”。

日志采样记录中有 `detect=572`、`hold=70`、`none=118`、`reject=28`。这些数字用于说明候选输出非常频繁，不能作为识别成功率。

## 2. 根因

当前 `cvlite_demo.py` 的入口是全画面 `cv_lite.grayscale_find_rectangles_with_corners()`；筛选条件只有候选矩形的长宽比和面积，并取最大面积候选。现有长宽比范围为 `0.52~0.92`，与靶纸及窗格都兼容；算法没有验证黑框、同心圆或红点等靶纸身份特征。

另外，当前仅采集灰度图，无法用红点做校验；`target_config.py` 也未被 `cvlite_demo.py` 导入，因此其中的红点阈值不参与当前流程。

时序逻辑亦需修正：连续确认阈值只影响缓存，不会阻止第一帧候选绘制和输出；跳变拒绝时只回退中心，不回退四角，导致 `reject` 日志中的中心与 corners 可能不属于同一候选。

## 3. 调研结论与方案决策

**明确结论：不要继续把“全画面矩形检测”作为主入口；应改为“黑框 ROI → ROI 内四边线 → 对角线交点 → 圆环/红点校验”的两阶段几何方案。**

该结论来自项目内已有调研：全画面 `find_rects()` 被判定会受局部边缘和背景干扰；“黑色连通域 ROI → ROI 内四边线 → 对角线交点”被列为推荐主方案。来源：选自 [靶心识别方案调研.md（第 14～18 行）](/C:/Users/zhntd/Desktop/DianJiXZ/firmware/K230_Target_Detection/research/靶心识别方案调研.md:14)。

推荐处理链路：

```text
RGB565 原始图像
  → 灰度黑色阈值 + find_blobs()，获得黑框候选 ROI
  → 仅在 ROI 内执行 find_lines()/find_line_segments()，恢复四条边
  → 相邻边求四角，两条对角线求交点，得到初始靶心
  → 依据四角做透视矫正
  → 在矫正图中校验同心圆；在中心小 ROI 检查红点
  → 连续确认后输出 valid、center_x、center_y
```

上述流程与现有靶心调研所列推荐实现路径一致。来源：选自 [靶心识别方案调研.md（第 20～27 行）](/C:/Users/zhntd/Desktop/DianJiXZ/firmware/K230_Target_Detection/research/靶心识别方案调研.md:20)。

### 为什么取对角线交点

靶纸是平面矩形，倾斜后投影为任意四边形。使用候选外接矩形中心会引入透视偏差；靶纸四角构成的两条对角线交点才对应其几何中心。若实物印刷圆心和纸张中心有固定偏移，应在正对相机时标定为常量补偿。来源：选自 [靶心识别方案调研.md（第 29～31 行）](/C:/Users/zhntd/Desktop/DianJiXZ/firmware/K230_Target_Detection/research/靶心识别方案调研.md:29)。

### cv_lite 的定位

保留 `cv_lite`，但只把它视作可选的性能加速工具，而不再单独负责靶纸身份判定。调研显示其底层是 Canny、轮廓拟合和矩形筛选，速度优势明显；但这类全画面矩形候选仍会遇到窗格等同形背景干扰。来源：选自 [2025-电赛E题-视觉方案调研.md（第 55～70 行）](/C:/Users/zhntd/Desktop/DianJiXZ/firmware/K230_Target_Detection/research/2025-电赛E题-视觉方案调研.md:55)。

## 4. 实施建议

### 阶段 A：先完成黑框 ROI

1. 保留 RGB565 原图，并从灰度图对暗色区域作阈值分割。
2. 使用 `find_blobs()` 找候选黑色连通域；按面积、长宽比、边界完整性筛选。
3. 候选 ROI 必须连续覆盖粗黑外框；窗格、桌面或背景黑物体不应直接成为有效靶。

本阶段验收标准来自现有调研：ROI 覆盖黑框；ROI 内能检测两近水平、两近竖直边；靶纸离开、黑框不完整或尺寸过小时输出 `valid=0`，不输出旧坐标。来源：选自 [靶心识别方案调研.md（第 32～34 行）](/C:/Users/zhntd/Desktop/DianJiXZ/firmware/K230_Target_Detection/research/靶心识别方案调研.md:32)。

### 阶段 B：四边几何与身份校验

1. 在 ROI 内检测边线，求相邻边交点得到四角，拒绝非凸、边长异常或方向不符合靶纸的候选。
2. 用两条对角线交点作为靶心初值。
3. 对四角做透视矫正，在统一坐标中检查黑色同心圆的数量、共心性和尺寸比例。
4. 在靶心附近的小 ROI 以 RGB 阈值检查红点；红点只用于确认或微调，不单独决定靶心。

全画面圆检测不作为主方案，因为靶纸倾斜时圆环会投影成椭圆；圆环应在已锁定靶框后作为二阶段校验。来源：选自 [靶心识别方案调研.md（第 14～17 行）](/C:/Users/zhntd/Desktop/DianJiXZ/firmware/K230_Target_Detection/research/靶心识别方案调研.md:14)。

### 阶段 C：时序与通信

1. 连续至少 3 帧同时通过“黑框 + 四边 + 圆环/红点校验”才发布 `valid=1`。
2. 跳变候选应整帧丢弃，恢复上一份完整有效状态或输出 `valid=0`；不得只回退中心而保留新 corners。
3. `hold` 只作为显示或极短暂平滑状态，不能作为控制系统长期使用的坐标来源。
4. UART 发送坐标时复用既有 `0xAA + x + y + area + XOR` 协议；无有效靶时令 `area=0`，使 MSPM0 明确区分“无目标”与“坐标 (0,0)”。来源：选自 [docs_learn/ques8.md（第 35～52 行）](/C:/Users/zhntd/Desktop/DianJiXZ/docs_learn/ques8.md:35)。

## 5. 采集与调参要求

- 相机避免直接面向窗户、灯管等强逆光背景；尽量锁定曝光和白平衡。
- 靶纸高度建议占有效画面约 1/4～1/2，黑框线宽至少保持 3～5 px。
- 建立包含靶纸远近、旋转、倾斜、强反光、空场、窗格、桌椅和局部遮挡的测试帧集。
- 每轮调参记录：黑色阈值、ROI 面积范围、边线阈值、圆环校验阈值、红点阈值、帧率和误检样例。

早期调研给出的二值化、形态学和面积阈值只能作为实机标定起点，不应直接视为通用固定值。来源：选自 [2025-电赛E题-视觉方案调研.md（第 30～53 行）](/C:/Users/zhntd/Desktop/DianJiXZ/firmware/K230_Target_Detection/research/2025-电赛E题-视觉方案调研.md:30)。

## 6. 验收目标

| 项目 | 目标 |
| --- | --- |
| 空场、窗格背景 | 不输出 `valid=1` |
| 靶纸正常入镜 | 连续有效率 ≥ 95% |
| 靶心稳定性 | 静止靶标抖动 ≤ 2 px |
| 靶纸离开/遮挡/黑框残缺 | 输出 `valid=0`，不沿用旧坐标作控制 |
| 性能 | 记录真实 FPS，并满足控制链路刷新需求 |

## 7. 是否训练模型

当前不训练 YOLO/KPU。靶纸的黑框、白底、圆环和红点具有固定几何结构，几何方法比检测框中心更适合输出高精度靶心。只有以下情况满足任一项时才进入模型方案：黑框经常遮挡；场地有大量相似黑色矩形且几何校验仍不能稳定排除；靶纸样式频繁变化。即使训练模型，模型也只用于给出 ROI，最终靶心仍由四角几何计算。来源：选自 [靶心识别方案调研.md（第 35～41 行）](/C:/Users/zhntd/Desktop/DianJiXZ/firmware/K230_Target_Detection/research/靶心识别方案调研.md:35)。
