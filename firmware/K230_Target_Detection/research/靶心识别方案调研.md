# 靶心识别方案调研

调研日期：2026-07-22  
调研人员：codex
适用对象：K230D CanMV、209.5 × 296 mm 黑框同心圆靶、640 × 480 实时画面。

## 问题与现场证据

目标是输出同心圆靶的几何靶心，而不是仅识别红点或目标框中心。

当前实拍帧保存在 `../pic/`。其中 `canmv-frame-2026-07-22T07-57-29-234Z.png` 显示：直接调用 `find_rects()` 会把中心圆附近的局部边缘误识别为小菱形，不能稳定产生外侧粗黑框四边形。运行日志也显示每帧候选矩形数在 0～14 间变化，经过筛选后大多数帧没有有效候选。

因此，“直接对整幅图调用 `find_rects()`”不作为主方案。

## 候选路线比较

| 路线 | 适用性 | 结论 |
| --- | --- | --- |
| 红点阈值/色块 | 红点近距离清晰时很快 | 仅作辅助；远距离、反光和红色圆环会误检。 |
| 全画幅 `find_circles()` | 靶面近似正对相机时可用 | 不作为主方案；靶面倾斜时圆环投影为椭圆。 |
| 全画幅 `find_rects()` | 细边框、高对比边缘 | 当前粗黑框实测误检中心圆，放弃作为入口。 |
| 黑色连通域 ROI → ROI 内四边线 → 对角线交点 | 黑框完整、背景干扰有限 | **推荐主方案**。先缩小搜索区域，再用直线恢复四个角，避免矩形 API 的局部误检。 |
| 黑框四角 → 透视矫正/同心圆校验 | 黑框四角已稳定 | 推荐作为第二阶段。对角线交点已足以得到靶面中心；圆环用于校验和近距离微调。 |
| KPU 目标检测模型 | 黑框经常遮挡、背景有相似黑框、靶纸样式不固定 | 后备方案。模型框中心不具备透视几何精度，仍需四角或圆环精定位。 |

## 推荐实现路径

```text
原始 RGB565 图像
  → 灰度图黑色阈值
  → find_blobs() 找黑色连通域，得到靶面 ROI
  → 在 ROI 内 find_lines()/find_line_segments() 找黑框四边
  → 由相邻边求四角，再取两条对角线交点
  → 在 ROI 内检查红色圆环是否围绕该中心
  → 输出 valid、center_x、center_y
```

### 为什么用对角线交点

靶纸和黑框在物理上是平面矩形。相机透视会使其在画面中成为任意四边形，直接取外接矩形的 `(x + w/2, y + h/2)` 会产生偏差；两条对角线的交点对应靶纸物理中心，更适合作为靶心基准。

前提：同心圆中心与黑框几何中心应固定重合。若实物存在固定印刷偏移，应在靶纸正对相机时测出该偏移并作为常量补偿。

### 第一阶段验收标准

1. 黄色 ROI 连续覆盖黑框，而不是桌面、支架或背景黑物体。
2. ROI 内检测到两条近水平、两条近垂直的边线。
3. 四角和对角线交点随靶子平移/转动连续变化。
4. 靶子完全离开画面、黑框不完整或尺寸过小时输出 `valid=0`，不输出旧坐标。

## 模型训练的决策条件

暂不训练。当前图片显示黑框和白色靶面具有固定外观，几何特征比模型输出框更适合获得靶心。

只有满足以下任一条件时才启动标注训练：

- 黑框被支架、手或其他物体频繁遮挡；
- 场地存在多个相似黑色矩形，连通域筛选无法稳定排除；
- 靶面类型、颜色或边框比例经常变化。

若训练，标注类别应为 `target_frame` 的外框，不标红点；训练集至少覆盖不同距离、旋转、俯仰、强反光、背景干扰和部分遮挡。模型仅用于给出 ROI，最终靶心仍由几何步骤计算。

## 公开参考

1. [piez/2025E](https://gitee.com/piez/2025E)：CanMV 实战代码。采用二值化、膨胀、矩形筛选、对角线交点，并讨论透视下不应直接取矩形中点。
2. [2025 电赛视觉实践记录](https://hongyiyang.online/%E7%94%B5%E8%B5%9B%E8%A7%86%E8%A7%89/)：使用 `find_blobs()` 获取黑色外框，再通过矩形四角和对角线交点求靶纸中心；还提出边缘贴合检查来排除背景干扰。
3. [2025-E题国一开源项目](https://oshwhub.com/yun_7783/waiting-for-express-delivery)：指出背景黑色物体会误识别，且相机与靶纸相对姿态变化时需要透视重构。
4. [CanMV K230 `find_blobs()` 官方示例](https://www.kendryte.com/k230_canmv/en/v1.3/example/omv/colortrack.html)：确认 K230 CanMV 支持 OpenMV 风格的色块/连通域检测 API。
