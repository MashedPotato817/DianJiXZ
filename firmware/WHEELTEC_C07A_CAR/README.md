# WHEELTEC C07A 基础巡线工程

基于 MSPM0G3507、WHEELTEC C07A 差速底盘和 TB6612 的低速基础巡线版本。当前目标是先稳定完成黑线直行、弧线回线和丢线搜线；十字路口、视觉、IMU 和赛题任务状态机不在本工程范围内。

## 当前行为

控制周期为 5 ms（200 Hz）。每个周期依次读取 8 路灰度传感器，计算黑线质心，再通过差速逆解和左右轮增量 PI 输出 PWM。

| 项目 | 当前值/行为 |
| --- | --- |
| 灰度线序 | 左侧物理传感器为通道 0，右侧为通道 7 |
| 灰度极性 | 黑线输出 `1`，白底输出 `0` |
| 基础巡线速度 | `45 mm/s` |
| 常规转向上限 | `0.40 rad/s` |
| 丢线搜线角速度 | `0.50 rad/s`（约 29°/s） |
| 丢线方向 | 沿最后一次有效偏线方向原地搜线 |
| 搜线终止 | 按命令角速度累计一圈（`2π`）仍未找到线则停车 |
| 首次上电全白 | 保持停车，不进入搜线 |

黑线偏右时，左轮目标速度提高、右轮目标速度降低，车辆向右以弧线回线；黑线偏左时相反。常规巡线阶段两轮保持前进，不采用右轮反转、左轮正转的原地转向。

丢线后的原地搜线会让左右轮反向低速旋转。其角速度由 `GRAY_LOST_SEARCH_ANGULAR_SPEED` 限制，避免给出高速自转指令。

## 使用方法

1. 使用 Keil uVision5 打开 `keil/empty_LP_MSPM0G3507_nortos_keil.uvprojx`。
2. 选择目标 `MSPM0G3507_Project`，编译并下载。
3. 上电后默认处于灰度巡线模式且电机停止；单击按键启停，双击在灰度巡线与 APP 遥控模式之间切换。
4. 初次测试必须先抬车确认：中间压线时两轮均向前；黑线移到左、右侧时车轮差速方向能使车辆回线。
5. 再放到直线黑胶带上低速试跑，确认后才逐级提高 `GRAY_BASE_SPEED_MM_S`。

OLED 会显示灰度原始读数、黑线位置、左右轮目标/实际速度以及电机启停状态，可用于现场确认传感器线序和闭环方向。

## 代码入口

- `empty.c`：系统初始化、中断使能和主循环显示/通信处理。
- `Control/control.c`：5 ms 控制中断、灰度巡线、丢线搜线、运动学与速度 PI。
- `Control/control.h`：底盘尺寸、编码器、巡线和搜线参数。
- `Hardware/`：TB6612、编码器、灰度选通、OLED、按键与 ADC 驱动。
- `docs/基础巡线三次审计_20260723.md`：当前实现的审计、验证记录和已知边界。

## 调参顺序

只在确认方向正确后调参，且一次只改一项：

1. `GRAY_BASE_SPEED_MM_S`：基础速度。先保持 `45 mm/s`，稳定后再提高。
2. `GRAY_STEER_GAIN`：回线灵敏度。拐弯冲出时小幅增大；出现左右摆动时减小。
3. `GRAY_MAX_ANGULAR_SPEED`：常规巡线的最大角速度。
4. `GRAY_LOST_SEARCH_ANGULAR_SPEED`：丢线原地搜线速度。若硬件抖动或搜线过猛，应先降低该值。

不要直接修改 `ti_msp_dl_config.c/h`。引脚或外设配置应在 `empty.syscfg` 中通过 TI SysConfig 修改后重新生成。

## 硬件与配置边界

- 当前参数按 MG513X `13 线 × 2 倍频 × 28:1` 配置，编码器分母为 `728`；更换或确认实际减速比后必须同步校准。
- 当前基础工程只使用 8 路数字灰度巡线。CCD、K230、JY62 等模块请在对应独立工程中继续开发，避免占用冲突的引脚或中断资源。
- 丢线的一圈限制按“命令角速度 × 时间”估算，不是 IMU 实测角度。

老师提供的 `../demo/` 用于参考 MSPM0G3507 外设驱动和工程结构，不是当前小车的直接编译目标。
