## 现象

实车出现持续向左转的现象。当前阶段只验证基础巡线，不需要十字路口直冲、丢线搜线或任务状态机。

## 审计结论

1. 灰度极性与实测定义一致：白底为 `0`，黑线为 `1`。
2. `Gray_Pos_mm` 的偏差符号、`Move_Z` 的负号以及 `Set_PWM(MotorA.Motor_Pwm, -MotorB.Motor_Pwm)` 与已恢复的 H 题工程一致，因此不在缺少单轮实测的情况下反转左右方向。
3. 先前将 `GRAY_STEER_GAIN` 从原有的 `1.70` 增至 `5.10` 没有实车依据，可能造成过度转向，现恢复为 `1.70`。
4. 原 `Gray_Mode()` 在全白时会以固定角速度原地搜线；若灰度未识别到黑线，车辆会持续单向旋转。十字直冲同样不属于当前基础验证范围。

## 当前最小控制逻辑

```text
读取 8 路灰度
  ├─ 没有黑线：左右轮目标速度清零，停车
  └─ 检测到黑线：质心求偏差 → 纯追踪转向 → 差速逆解
```

## 历史日志补充结论

根目录历史记录 `docs/巡线速度与逻辑评估_20260710.md` 表明：

1. 旧底盘的高速三档版本曾以 `GRAY_STEER_GAIN = 1.70` 编译通过并实车调试，因此本轮将无实车依据的 `5.10` 恢复为 `1.70`。
2. 该记录把“原始低速稳定版”（约 `45 mm/s`）定位为排查硬件接线、灰度线序、电机方向和闭环方向的专用版本。
3. 当前底盘已进行硬件重构，`150 mm/s` 只作为基础逻辑的保守运行值，仍不适合作为方向问题的首轮排查速度。

### 下一轮建议

在确认左右转向方向前，将 `GRAY_BASE_SPEED_MM_S` 临时降为 `45 mm/s`；完成下列检查后，再恢复到 `150 mm/s` 并逐级提速：

1. 中间两路检测黑线时，左右轮应同向前进。
2. 黑线分别移至传感器左、右侧时，车轮差速方向应与回线方向一致。
3. 黑线完全离开传感器阵列时，车辆应停车，不能自行单向旋转。

## 实车验证顺序

1. 抬车启动，确认线在中间两路时左右轮同向、速度接近。
2. 手动将黑线移到传感器左侧、右侧，分别确认车轮差速方向与预期一致。
3. 放到直线黑胶带上，以 45 mm/s 验证不持续偏转。
4. 只有在以上三项通过后，才提升至 150 mm/s；丢线处理、变速或路口选线留待后续审计。
