## 最高优先级：转向符号未闭环确认

当前 `Gray_Mode()` 使用：

```c
Move_Z = GRAY_STEER_GAIN * Move_X * curvature;
```

而根目录归档的高速三档版本及 `WHEELTEC_C07A_CAR_JY62_GapBridge` 均使用负号：

```c
Move_Z = -GRAY_STEER_GAIN * Move_X * curvature;
```

结合当前差速逆解：

```c
MotorA.Target_Encoder = Vx - Vz * Wheelspacing / 2.0f;
MotorB.Target_Encoder = Vx + Vz * Wheelspacing / 2.0f;
```

若 `Gray_Pos_mm` 的编号确实是“通道 0 在车体左侧、通道 7 在车体右侧”，则黑线落在右侧会得到正偏差；当前正号会让右轮目标更快，车辆向左转，属于远离右侧黑线的方向。该链路与“疯狂左转”现象相符。

但 3-8 选通器的通道编号是否与传感器物理左右方向一致尚未实测。因此，本轮**不直接修改符号**，必须先完成下列静态传感器测试：

1. 只遮挡最左侧物理传感器，记录 `Gray_Data[0..7]` 中为 `1` 的编号。
2. 只遮挡最右侧物理传感器，记录 `Gray_Data[0..7]` 中为 `1` 的编号。
3. 若左侧为低编号、右侧为高编号，应恢复 `Move_Z` 前的负号；若编号相反，则保留正号并调整坐标数组定义。

## 低速 PI 的测量分辨率风险

按当前轮径 76.8 mm、周长约 0.2413 m、编码器分母 780、控制周期 5 ms 计算：

```text
45 mm/s 对应每个控制周期约 0.73 个编码器计数
1 个计数对应的瞬时速度约 61.9 mm/s
```

因此在 `45 mm/s` 下，单周期速度反馈会在 `0` 与约 `61.9 mm/s` 之间跳变。该速度用于确认方向是合适的，但不适合据此评价或细调 `Velocity_KP=400`、`Velocity_KI=300`；PI 参数应在方向正确且速度提高后，结合多周期平均速度再调。

## 其他发现

1. 灰度读取每次执行 8 个通道、每个通道等待 50 us，总等待约 400 us，低于 5 ms 控制周期，当前可接受。
2. `TIMER_0_INST_IRQHandler()` 内层使用 `if (DL_TIMER_IIDX_ZERO)` 常量式判断，而不是读取实际中断索引。外层已检查定时器存在挂起中断，现阶段不是左转根因；后续整理中断时应改为显式读取并判断中断索引。
3. `GRAY_SENSOR_SPAN_MM = 85` 被解释为 8 个传感器中心的总跨距。若 85 mm 实际是模块外形长度而非中心跨距，只会改变偏差与转向量级，不会单独导致固定向左转。

## 本轮结论与下一步

1. 保持 `45 mm/s`、`0.40 rad/s` 和丢线停车，不恢复复杂巡线状态机。
2. 先完成左右物理传感器与通道编号的对应测试，再决定 `Move_Z` 的最终符号。
3. 转向符号确认后，先跑直线；直线稳定后再提高速度并评估 PI。
