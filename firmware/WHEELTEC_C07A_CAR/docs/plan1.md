# 让小车重新跑起来 —— 基础巡线验证计划

日期：2026-07-23

## 背景

硬件重构后的差速小车，代码已精简为最小巡线闭环（质心→曲率→差速），但实车出现**持续左偏**。审计文档（`基础巡线审计.md`）明确指出当前 150 mm/s 太快，不适合排查方向问题，应先降到 45 mm/s。

电机/编码器未更换，`780` CPR 系数仍然有效。

## 当前代码结构回顾

```
Timer ISR (200Hz):
  encoder计数 → Gray_Mode() → 曲率转向 → 运动学逆解 → 增量PI → Set_PWM(A, -B)
```

- `Gray_Mode()`: 质心加权 → `curvature = 2y/(L²+y²)` → `ω = -1.70 * Vx * curvature`，ω 钳位 ±1.20 rad/s
- OLED 已显示：传感器原始值 `G01010110`、线位置 `P+042`、角速度、左右轮目标/实际速度
- 丢线=停车，无十字直冲，无分档变速

## 代码改动

### 1. 降速到 45 mm/s（审计建议的最低排查速度）

**文件**: `firmware/WHEELTEC_C07A_CAR/Control/control.h`

```c
#define GRAY_BASE_SPEED_MM_S  45.0f   // 从 150.0f 临时降低，用于方向排查
```

### 2. 按比例缩小最大角速度

```c
#define GRAY_MAX_ANGULAR_SPEED  0.40f  // 从 1.20f 按速度比例缩小
```

> 注：曲率转向公式 `ω = -Gain * Vx * curvature` 本身会随 Vx 线性缩放，钳位值不太可能被触发。此处降低钳位是防御性的。

### 不改动的地方

- **不碰符号约定**：`Move_Z = -GAIN * Vx * curvature` 和 `Set_PWM(A, -B)` 与历史 H 题工程一致，审计已确认
- **不加新代码**：OLED 已经显示所有关键诊断数据（传感器、线位置、角速度、左右轮速度）
- **不恢复高级功能**：十字直冲、丢线搜线、分档变速留待方向确认后再加

## 实车验证清单（按顺序）

### 第0步：抬车空转 — 确认基本运动

车架架空，车轮悬空，按键启动：

| 检查项 | 预期结果 | OLED 怎么看 |
|--------|----------|------------|
| 线在中间两路（传感器3/4） | 左右轮同向、速度接近 | `L+045` ≈ `R+045`，符号相同 |
| 线移到最左侧（传感器0/1） | 左轮减速/反转，右轮加速，车向右转 | `L` 变小/变负，`R` 变大 |
| 线移到最右侧（传感器6/7） | 右轮减速/反转，左轮加速，车向左转 | `R` 变小/变负，`L` 变大 |
| 线完全离开传感器 | 两轮立即停止 | `ON` 但 `L0000 R0000` |

**如果差速方向反了**：说明硬件重构后电机/编码器接线顺序与预期不同，需要调符号——但审计说符号与 H 题工程一致，这个可能性较低。

### 第1步：放到直线上 — 确认不持续偏转

地上贴黑色胶带（电工胶布即可），放车上线：

| 检查项 | 预期结果 |
|--------|----------|
| 45 mm/s 走直线 | 不持续偏左或偏右，蛇形摆动可接受 |
| 稳定后 OLED `P` 值 | 在 ±5mm 内波动 |
| 稳定后左右轮速度 | `L` ≈ `R`，长期平均值一致 |

**如果仍然持续左偏**：
- 看 OLED 左右轮实际速度是否相等 → 如果左轮实际速度明显高于右轮，说明机械/电气不对称
- 看灰度传感器 `Gxxxxxxxx` 是否线偏左（即线在中间时传感器显示偏左）→ 传感器物理安装偏移
- 如果传感器显示正确但车轮差速不对称 → PI 控制问题，可能需要调 KP/KI

### 第2步：简单弯道测试

在地上贴 S 形或圆弧弯道：

| 检查项 | 预期结果 |
|--------|----------|
| 左弯 | 车能跟随左转，不掉线 |
| 右弯 | 车能跟随右转，不掉线 |
| 掉线后 | 立即停车，不旋转 |

### 第3步：逐级提速

方向确认无误后，依次测试：

```
45 mm/s ✅ → 80 mm/s → 120 mm/s → 150 mm/s
```

每级都在直线上验证不偏转。如果某级开始偏转，记录该速度，回到上一级。

## 待确认问题

1. **硬件重构具体改了哪些？** 电机没换，但轮距变了（130→178mm），轮径变了（67→76.8mm），灰度传感器前视距离变了（150→260mm）。这些参数已更新到 control.h，但需要实车验证。
2. **左偏是否是 PI 增益过高导致的振荡？** KP=400，KI=300 在低速 45 mm/s 下可能过高，如果出现抖动可以尝试减半。
3. **灰度传感器能否正确检测线？** 用 OLED 第一行 `Gxxxxxxxx` 确认每个传感器在黑线上正常翻转。

## 成功标准

- 45 mm/s 直线上不持续偏左（允许蛇形摆动）
- 手动移线时差速方向正确
- 无黑线时停车
- 逐级提速至 150 mm/s 后仍稳定
