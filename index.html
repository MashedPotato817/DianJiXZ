Key Insight

一句话结论

控制题已经从"让车跑起来、让电机转起来"发展为"感知、定位、运动控制、任务状态机、通信和系统可靠性综合考核"。小车仍是高频载体，但越来越强调视觉、超声定位、自动瞄准、避障、巡查、协同作业和多传感器融合。

MSPM0 + TB6612 灰度模块 K210 视觉 热成像 超声定位 编码器闭环 PID 控制 状态机 多传感器融合

Deep Dive

各年份题目分析

2019 — 小车与基础控制

任务物理性强，机械结构、电源和基础闭环决定成败。

A题

电动小车动态无线充电

  • 小车沿指定路线稳定运行
  • 接收端与发射端有效耦合
  • 动态运动中供电稳定
B题

巡线机器人

  • 灰度/视觉循迹
  • 弯道控制 + 停车定位
  • 交叉口、标志点识别
H题

模拟电磁曲射炮

  • 角度控制精度
  • 目标距离和角度估计
  • 机械重复性 > 算法复杂度

2021 — 任务型机器人和闭环增强

开始强调"任务流程"，不只是跑路线，还要完成配送、识别、能量管理。

F题

智能送药小车

  • 清晰状态机：待命→识别→寻路→到站→执行→返回
  • 显示屏/串口日志判断当前状态
G题

植保飞行器

  • 飞行稳定性
  • 传感器融合和保护逻辑

2023 — 感知定位和协同系统

明显提高系统综合度——目标跟踪、声学定位、空地协同、低摩擦平台。

E题

运动目标控制与自动追踪

  • 摄像头/K210/OpenMV + 云台
  • 追踪要处理延迟，不能只靠简单阈值
F题

声学智能定位

  • 核心是标定和误差建模
  • 定时器捕获、同步、滤波比花哨算法重要
G题

空地协同消防

  • 通信协议要简单可靠
  • 多设备系统必须有主从状态和超时处理

2025 — 小型智能系统

更像"小型智能系统"，很多任务不再单独考电机，而是考"感知后如何动作"。

E题

简易自行瞄准装置

  • 云台PID + 图像坐标到角度映射
  • 机械结构刚性直接影响得分
H题

野生动物巡查

  • K210/视觉模块 + 数据记录
  • 路线完成 + 识别准确 + 结果显示
K题

自动避障小车

  • 超声/红外/ToF/视觉检测障碍
  • 高分关键："能稳定绕过去"，不是传感器越多越好

Roadmap

备赛优先级

1

第一阶段

目标：保证基本分
  • 小车直行不明显跑偏
  • 稳定低速巡线
  • 定距停车
  • 识别障碍并停车
  • 串口或屏幕显示状态
  • 每次上电行为一致

验收：连续10次≥8次成功；电池降压后仍运行

2

第二阶段

目标：提高发挥分
  • 编码器速度闭环
  • 弯道减速、直道加速
  • 路口/任务点状态机
  • 目标识别串口协议稳定
  • 关键参数可现场调整

验收：同路线耗时稳定；停车误差可调可测

3

第三阶段

目标：现场鲁棒性
  • 灰度阈值自动/半自动标定
  • 摄像头曝光/阈值预案
  • 电源抗干扰
  • 所有线束固定
  • 备份程序和参数

验收：改变光照后仍完成基础任务；大电流不复位

建议开发顺序

  1. TB6612 + 编码器 → 稳定底盘
  2. 灰度巡线 + 定点停车
  3. K210 → 目标识别/标志识别
  4. 热成像/超声 → 人体/热源/障碍检测
  5. 统一串口协议 → 避免临场调不出来
Watch Out

常见失分点

小车跑偏
原因：左右电机差异、轮胎摩擦不同
→ 编码器闭环、速度标定
弯道冲出
原因：速度太高、控制滞后
→ 弯道减速、偏差限幅
停车不准
原因：只按延时停车
→ 编码器定距、目标点二次校正
目标识别不稳
原因：光照变化、阈值固定
→ 自动曝光/阈值标定、增加补光
通信丢包
原因：电机干扰、协议无校验
→ 帧头、长度、校验、超时重传
现场复位
原因：电源压降、地线布置差
→ 动力和主控分供电、共地、滤波
调试困难
原因：无显示、无日志
→ OLED + 串口输出关键状态
临场参数不合适
原因：实验室和现场环境不同
→ 参数可调、保存默认值
Training

备赛训练题单

按优先级排序

  1. 小车直行 1m、2m、3m 定距停车
  2. 原地左转/右转 90°、180°
  3. 灰度循迹跑直线、弯道、十字路口
  4. 路口计数并在指定路口停车
  5. 遇障停车、绕障、回到路线
  6. K210 识别颜色块/二维码/数字，UART 发给主控
  7. 热成像识别热源方向，主控根据方向转向
  8. 完整状态机：“出发-识别-到点-执行-返回”
Template

推荐代码框架

主循环结构

int main(void)
{
    board_init();
    motor_init();
    encoder_init();
    line_sensor_init();
    vision_init();

    task_state = TASK_WAIT_START;

    while (1) {
        sensor_update();
        vision_update();
        task_update();
        control_update();
        debug_update();
    }
}

中断里只做短任务

void timer_10ms_irq(void)
{
    encoder_sample();
    control_tick = 1;
}

void uart_rx_irq(void)
{
    push_byte_to_ring_buffer();
}
🚫 中断里不要做：大量 printf | 长延时 | 图像解析 | 复杂路径规划 | 阻塞等待串口
